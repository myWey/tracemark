# Map Generators

> What goes under `.agent/map/`, who produces it, when, and how to detect
> staleness.

## Files

| File | Generator workflow | Sources | When to regenerate |
|---|---|---|---|
| `architecture.md` | `workflows/regenerate-map.md` | `package.json`, `apps/`, `packages/` deps | After dep change or new package |
| `architecture.svg` | `workflows/regenerate-map.md` | import graph | Same as above |
| `api-surface.md` | `workflows/regenerate-map.md` | `shared/api-contracts/`, `shared/schemas/` | After contract change |
| `component-tree.md` | `workflows/regenerate-map.md` | `packages/ui-*`, `apps/*` | After UI change |
| `route-map.md` | `workflows/regenerate-map.md` | `apps/*/routes` or framework router | After route change |
| `data-flow.svg` | `workflows/regenerate-map.md` | `shared/events/`, state machines | After event/state change |
| `adr-timeline.md` | `workflows/regenerate-map.md` | `.agent/adr/*` | After any ADR change |

## Header convention

Every map file starts with:

```
<!-- Generated by: .agent/workflows/regenerate-map.md
     At: {ISO timestamp, FYI only}
     Source-hash: sha256:{hex}
     Sources: [paths] at commit {git sha}
     Do not hand-edit. Run the generator. -->
```

## Staleness check

The header `Source-hash` is the source of truth for "is this file fresh".
Compute the hash over the listed `Sources` at HEAD. If it differs, the file
is stale.

## Hooks

- `post-merge` git hook OR IDE hook: regenerate any map file whose
  `Source-hash` no longer matches.
- `post-task-execution` IDE hook: same.
- Agent on session start: if any map header is stale, run the generator
  before doing other work.
