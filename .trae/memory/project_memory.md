# Project Memory: agentOS

> Cross-session project-level conventions and decisions for the agentOS workspace.

## Conventions

- Specs are generated by Trae native `/spec` into `.trae/specs/<change-id>/` (spec.md + tasks.md + checklist.md).
- Root-level `specs/` and `plans/` directories have been removed to avoid dual-path confusion.
- Project asset files under `docs/` are templates until `/onboard` runs; they display a status banner at the top.
- Build/test/lint/typecheck entrypoints live in `scripts/*.sh`; in skeleton mode they exit 0 with an explanatory message.
- Hooks follow the Trae/Claude Code protocol: stdin JSON (`HookInput`), stdout JSON decisions, exit code 0/2.
- Always-on rules are intentionally lean (~70 lines): `00-core-principles.md` + `01-conventions-and-safety.md`.
- Context-heavy methodologies (memory, context engineering) are loaded as skills, not rules.

## Architecture Decisions

- Keep `.trae/rules/*.md` as the rule location instead of migrating to `.trae/project_rules.md`, because the existing structure is already recognized by Trae and migration risk outweighs benefit.
- Use self-contained bash scripts for hooks rather than a shared helper library, to keep each hook independently testable.
- Output structured JSON decisions to stdout on PreToolUse/Stop block events, plus bilingual human-readable feedback to stderr.

## Lessons

- Full-width Chinese punctuation next to `$variable` triggers `set -u` errors on macOS bash 3.2. Always use `${variable}` in bilingual hook strings.
- Hook scripts must read JSON from stdin; relying on environment variables or arguments is not portable across Trae/Claude Code.
- PostToolUse formatters must not silently swallow failures (`2>/dev/null || true`); failures should be reported so the agent can react.
