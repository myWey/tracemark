# AGENTS.md

> The README for agents. This file is the single source of truth for any
> AI coding agent working in this repository.
> Kept deliberately lean — only information agents cannot discover on
> their own.

## Project Positioning

This repository hosts **TraceMark** — a local-first macOS screenshot annotation
menu-bar app, embedded inside the agentOS workflow harness. Agents should use
the docs in `docs/` and the commands below as the single source of truth for
building, testing, and shipping TraceMark.

## Non-Negotiable Workflow

Every non-trivial task must follow the 6-phase closed loop. Skipping
phases is the #1 cause of misaligned agent output.

```
Research → Spec → Plan → Execute → Review → Ship
```

### Integration with Trae Native Mechanisms

| Phase | Trae Native Command | Output | Location |
|------|-------------|--------|------|
| Research | (explorer subagent) | context summary | in conversation |
| Spec | `/spec` (native) | `spec.md` + `tasks.md` + `checklist.md` | `.trae/specs/<change-id>/` |
| Plan | `/plan` (native) | plan document | `.trae/specs/<change-id>/` |
| Execute | (executor subagent) | code + commits | git history |
| Review | `/review` (custom) | review report | in conversation |
| Ship | `/ship` (custom) | ship summary | in conversation |

> **Note**: `/spec` and `/plan` use Trae native commands — do not override
> them with custom versions. The native commands auto-generate the three
> files into `.trae/specs/<change-id>/`.

### Iron Laws (Strict Mode)

1. **No code without a spec**: tasks over 50 lines or spanning 2 files
   must run `/spec` first.
2. **No plan without research**: tasks touching unfamiliar modules must
   be researched first.
3. **No ship without review**: nothing reaches `/ship` without passing
   the `/review` checklist.
4. **Every checkpoint must commit**: provides a rollback point.
5. **When unsure, ask**: never silently assume.
6. **Goal-driven execution**: turn tasks into verifiable goals and loop
   until verification passes.

## Document Framework (5 Layers)

```
L1 Strategy:     docs/PHILOSOPHY.md, docs/ROADMAP.md         → "Why" (stable)
L2 Architecture: docs/ARCHITECTURE.md, docs/TERMS.md, UI-UX  → "What" (on-demand)
L3 Standards:    .trae/rules/*.md                            → "How" (always-on)
L4 Execution:    .trae/specs/<id>/{spec,tasks,checklist}.md  → "This task" (per-task)
L5 Decisions:    docs/adr/, docs/rfc/                        → "Why this choice" (on-demand)
```

**Rule**: upstream doesn't reference downstream. SSOT within each layer.
**Harness** (.trae/): controls workflow (rules, skills, agents, hooks). Generic, embeddable.
**Project** (docs/): project-specific guidance. Generated by `/onboard`.

```
agentOS/
├── AGENTS.md, CLAUDE.md          # Entry points
├── .trae/                         # Harness (rules, skills, agents, hooks, specs/)
├── docs/                          # Project (L1-L2 + L5 documents, workflow-guide)
├── eval/                          # Smoke tests (run-smoke.sh)
└── scripts/                       # build.sh, test.sh, lint.sh, typecheck.sh
```

See `docs/workflow-guide.md` for the full dependency graph and layer design principles.

## Coding Conventions

- **Language**: match surrounding code. For new files, follow the
  dominant language of the nearest module.
- **Naming**: see `.trae/rules/01-conventions-and-safety.md`.
- **Imports**: prefer path aliases (`@/`) when the project configures
  them; otherwise use relative imports.
- **Comments**: only add them when the logic is not self-evident. Never
  restate code.
- **Error handling**: validate at system boundaries (user input, external
  APIs). Trust internal code.

## Context Hygiene

- **Subagents handle exploration**: never dump 30 file reads into the
  main conversation — delegate to an explorer subagent and keep only
  the summary.
- **Skills load on demand**: the full body loads only when the
  description matches the task.
- **Rules are always-on**: keep `.trae/rules/` lean — every line costs
  tokens every turn.
- **Hooks auto-inject**: the SessionStart hook auto-injects project
  state to enable cross-conversation continuity.
- **When to start a new conversation**: when switching features, after
  shipping a checkpoint, or when context drift causes contradictions.

## Cross-Conversation Continuity

Context is finite; continuity relies on persisted spec files in `.trae/specs/`, the SessionStart hook (auto-injects git/spec/memory state), and `.trae/memory/`. See the `context-engineering` and `memory-management` skills for details.

See `.trae/rules/00-core-principles.md` for the full set of agent behavior principles and failure mode countermeasures.

## How to Use This Environment

1. **Small tasks (< 50 lines, < 2 files)**: skip the spec, plan briefly, then execute.
2. **Medium tasks**: `/spec` → `/plan` → execute → `/review` → `/ship`.
3. **Large tasks / new systems**: full 6-phase closed loop, with a subagent for each phase.
